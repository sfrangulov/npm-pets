# npm-pets.dev Site (v0.4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Public Next.js site at `npm-pets.dev` with profile pages (`/u/<user>`), SVG card endpoint (`/card/<user>.svg`) for README embedding, and PNG OG image endpoint.

**Architecture:** Two side-by-side packages. `npm-pets/` (existing CLI) gets a library entrypoint exporting `buildProfile` and `formatCard` so the site reuses the same rendering pipeline. `web/` is a new Next.js 15 App Router project that depends on `npm-pets` (via `file:../npm-pets` for local dev, npm registry for Vercel). Routes: `/` (landing), `/u/<user>` (server-rendered profile page with OG meta), `/card/<user>.svg` (SVG endpoint), `/og/<user>.png` (PNG endpoint). Per-route `Cache-Control` headers handle edge caching.

**Tech Stack:** Next.js 15 (App Router), Node runtime (not Edge — needed for `@resvg/resvg-js` native binding), `npm-pets` workspace dep, Tailwind CSS for landing/profile pages.

**Spec:** `docs/specs/2026-05-07-wow-features-roadmap.md` § 6.

## Task 3: Profile cache helper

**Files (in `web/`):**
- Create: `lib/profile-cache.ts`

- [ ] **Step 1: Implement an in-memory TTL cache wrapping `buildProfile`**

Create `lib/profile-cache.ts`:

```ts
import { buildProfile, type Profile } from "npm-pets";

interface Entry { profile: Profile; expiresAt: number; }
const cache = new Map<string, Entry>();
const TTL_MS = 30 * 60 * 1000; // 30 minutes
const inflight = new Map<string, Promise<Profile>>();

export async function getProfile(target: string): Promise<Profile> {
  const key = target.toLowerCase().trim();
  const now = Date.now();
  const hit = cache.get(key);
  if (hit && hit.expiresAt > now) return hit.profile;

  const existing = inflight.get(key);
  if (existing) return existing;

  const token = process.env.GITHUB_TOKEN;
  const promise = (async () => {
    try {
      const profile = await buildProfile({
        target: key,
        type: "auto",
        token,
        cache: undefined,
        concurrency: 4,
      });
      cache.set(key, { profile, expiresAt: Date.now() + TTL_MS });
      return profile;
    } finally {
      inflight.delete(key);
    }
  })();
  inflight.set(key, promise);
  return promise;
}

export function profileCacheStats() {
  return { size: cache.size };
}
```

Two layers: TTL cache (30 min) + in-flight de-dupe so concurrent requests for the same user don't fan out to npm/GitHub multiple times.

## Task 4: SVG card endpoint

**Files (in `web/`):**
- Create: `app/card/[user].svg/route.ts`

Note: Next.js App Router treats the literal `[user].svg` segment as a dynamic route parameter `user` with a `.svg` suffix. The file path must be exactly `app/card/[user].svg/route.ts`. Inside the handler, the param value is `user` without the `.svg`.

- [ ] **Step 1: Implement the handler**

Create `app/card/[user].svg/route.ts`:

```ts
import { NextResponse } from "next/server";
import { formatCard } from "npm-pets";
import { getProfile } from "@/lib/profile-cache";

export const runtime = "nodejs";
export const dynamic = "force-dynamic";

interface RouteContext { params: Promise<{ user: string }>; }

export async function GET(_req: Request, { params }: RouteContext) {
  const { user } = await params;
  const target = decodeURIComponent(user).replace(/^@/, "");

  let profile;
  try {
    profile = await getProfile(target);
  } catch (err) {
    return NextResponse.json(
      { error: (err as Error).message || "failed to build profile" },
      { status: 404 },
    );
  }

  const svg = await formatCard(profile, 3);
  return new Response(svg, {
    status: 200,
    headers: {
      "Content-Type": "image/svg+xml; charset=utf-8",
      "Cache-Control": "public, max-age=300, s-maxage=1800, stale-while-revalidate=86400",
    },
  });
}
```

Headers explained:
- Browser cache: 5 min (lets a user refresh and see slightly stale data fast).
- CDN/edge cache: 30 min (Vercel's `s-maxage`).
- Stale-while-revalidate 24 h: serve stale while regenerating in background.

export const runtime = "nodejs";

    headers: {
      "Content-Type": "image/png",
      "Cache-Control": "public, max-age=300, s-maxage=1800, stale-while-revalidate=86400",
    },
  });

**Risks flagged:**
- `@resvg/resvg-js` is a native binding. Vercel's serverless function builders need it pre-built for Linux x64. The `serverExternalPackages: ["@resvg/resvg-js"]` setting in `next.config.mjs` (Task 2) tells Next.js to leave it out of the bundle so the native binary loads at runtime. If deploy fails with native-binding errors, that's the lever.
- The `npm-pets` workspace dep (`file:../npm-pets`) works for local dev but Vercel deploy needs the published package. The README (Task 8) documents the swap.
- Inter font in `formatCard` is loaded via `readFileSync(import.meta.url, ...)`. When `npm-pets` is consumed as a published package, the path resolves into `node_modules/npm-pets/dist/assets/fonts/`. The `loadFonts` candidate-path fallback in `npm-pets/src/formatters/card.ts` should handle this — verify after first deploy. If broken, add another candidate path or change the loader to use `import.meta.resolve`.
